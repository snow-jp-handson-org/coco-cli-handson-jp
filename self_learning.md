# 自由試行シナリオ集

ハンズオン本編で学んだ機能を活かして、自由に Cortex Code を試してみましょう。
以下から興味のあるシナリオを選んで取り組んでください（複数OK）。

---

## A. データ分析（★☆☆）

MART_SALES に対して自分の業務で使いそうな分析クエリを自然言語で試す。

```
#MART_SALES で曜日別の売上傾向を分析して、最も売上が落ちる曜日の改善提案を出して
```

---

## B. アプリケーション開発（★★☆）

Step 8 の Streamlit を自分好みにカスタマイズ、または全く新しい UI を作る。

```
売上データをリアルタイムで監視するダッシュボードをStreamlitで新規作成して。KPIカード・アラート閾値設定・期間比較を含めて
```

---

## C. エージェント開発（★★☆）

Step 6 で作った Agent を拡張、または新しいユースケースの Agent を作る。

```
MART_SALESとCUSTOMER_REVIEWSを使って、「商品の評判が悪化したら自動で売上影響を算出する」Cortex Agentを作って
```

---

## D. Skill開発 ― 独自ワークフロー（★★★）

自部署の定型業務を Skill 化する（例: 週次報告、異常検知、顧客セグメント）。

```
毎週月曜に実行する「先週の異常値検知レポート」を生成するSkillを作って。前週比で売上が30%以上変動した商品を抽出し、原因仮説をつけて
```

---

## E. データパイプライン構築（★★☆）

Dynamic Table + データ品質チェック + アラートの組み合わせで自動化パイプラインを作る。

```
MART_SALESから「日次KPIサマリ」Dynamic Tableを作り、NULL率が5%を超えたらSlack通知するAlertも設定して
```

---

## F. ドキュメント自動生成（★☆☆）

今日作ったテーブル・Dynamic Table・Semantic View の技術仕様書を CoCo に自動生成させる。

```
SNOWRETAIL_DB.SNOWRETAIL_SCHEMA の全オブジェクト（テーブル、Dynamic Table、Semantic View）の技術仕様書をマークダウンで作成して。各オブジェクトのカラム定義・リレーション・用途を含めて
```

---

## G. テストデータ生成 + 品質検証（★★☆）

CoCo にテストデータのシナリオを考えさせ、データ品質チェック Agent と組み合わせて検証する。

```
MART_SALESに「売上がNULL」「日付が未来」「数量がマイナス」の異常レコードを含むテストデータを10件生成して、data-quality-checker で検出できるか確認して
```

---

## H. 自社データ持ち込み分析（★☆☆）

参加者が自分のアカウントにあるテーブルや CSV を CoCo で探索・分析する。

```
自分のSnowflakeアカウントに接続して、使えるテーブル一覧を見せて。その中から一番行数が多いテーブルの概要と改善提案を出して
```

---

## J. プロンプトエンジニアリング比較（★☆☆）

同じ分析依頼を「曖昧なプロンプト」と「構造化プロンプト」で比較し、AGENTS.md / Skill の価値を体感する。

```
「売上どう？」と「#MART_SALES の直近3ヶ月のチャネル別・カテゴリ別クロス集計を、前月比成長率付きで表形式で出して」の2つを実行して、出力の違いを比較して
```
